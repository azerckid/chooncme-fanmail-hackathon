# 답메일 지침 고도화: LangGraph 활용 검토

## 1. 목적

현재 답장 생성은 단일 LLM 프롬프트(`lib/llm/reply-prompt.ts`)로 이루어진다. 지침을 더 잘 지키고, 상황에 맞는 답장을 만들기 위해 **LangGraph**를 이용한 다단계 파이프라인 도입 가능성을 검토한 의견을 정리한다.

---

## 2. LangGraph로 할 수 있는 것

### 2.1 다단계 파이프라인

한 번의 큰 프롬프트 대신, 그래프로 단계를 나눌 수 있다.

| 단계 | 역할 |
|------|------|
| 1 | 팬레터 분석: 주제·감정·언어 추출 |
| 2 | 톤·강조할 지침 결정 (안부 질문 유무, 노래/춤 구체화 수준 등) |
| 3 | 초안 생성 |
| 4 | 지침 준수 점검 (질문 1~2개, 250자 이상, 금기 표현 등) |
| 5 | 미달 시 해당 부분만 수정 후 최종 출력 |

지침을 단계별로 적용·검증할 수 있다.

### 2.2 조건 분기

- 팬이 슬픈 이야기를 했으면 → 위로 톤 + "슬픈일은 없으셨나요?" 등 강조
- 일상/재미 이야기가 많으면 → "재미있는 일은 없으셨나요?" 등 연결

상황에 따라 강조할 지침을 분기로 선택할 수 있다.

### 2.3 지침을 서브그래프로 분리

- "안부 질문 1~2개 고르기" 전용 노드: 팬레터 요약 → 적절한 질문 후보 출력
- "노래/춤 구체 에피소드 하나 고르기" 전용 노드: 후보에서 하나 선택

지침을 작은 단위로 나누어 수정·테스트가 쉬워진다.

### 2.4 단계별 출력 스키마 고정

각 단계 출력을 Zod 등으로 고정하면, "질문 개수", "노래/춤 구체 언급 유무" 등을 코드로 검사하기 쉽다.

## 3. 고려할 점

| 항목 | 내용 |
|------|------|
| 비용·지연 | 답장 한 통당 LLM 호출이 1회 → 3~5회로 늘어날 수 있음. 10~30분 지연 창 안에 넣을지 검토 필요. |
| 복잡도 | 그래프 정의, 상태 설계, 에러/재시도 처리 증가. |
| 효과 대비 | "지침을 조금 더 잘 지키게"만 목표라면, 프롬프트 다듬기나 2단계 프롬프트(계획 → 작성)만으로도 상당 부분 개선 가능. |

---

## 4. 의견 정리

- **고도화할 여지가 있다.** LangGraph를 쓰면 지침을 단계·분기·검증으로 쪼개어 적용하는 형태로 체계화할 수 있다.
- **도입 순서 제안**
  1. 당장은 LangGraph 없이, **지침 문구 보강 + 2단계 프롬프트(계획 → 작성)** 로 개선해 본다.
  2. "지침을 더 정교하게 지키고 싶다"가 명확해지면, **분석 → 지침 선택 → 초안 → 검증 → 수정** 그래프를 LangGraph로 도입한다.
- **구조 제안**: LangGraph를 쓸 경우, 답장 생성만 Node 서비스나 별도 워커로 분리하고, email-worker는 그 서비스를 호출하는 쪽으로 두면 구조가 깔끔해진다.

## 5. 검토 의견 (2026-02-06)

### 현재 프롬프트 개선 현황

문서 작성 이후, `reply-prompt.ts`에 다음 지침이 추가됨:

- **노래/춤 구체화**: 발성, 호흡, 비브라토, 웨이브, 팝 등 구체적 예시 명시
- **질문 예시 확장**: "재미있는 일은?", "슬픈 일은?" 등 다양한 안부 질문 추가
- **공식 이름 언어별 고정**: 한국어(춘심이), 영어(ChoonCme), 일본어(春心)

### 검토 결과

| 항목 | 의견 |
|------|------|
| 문서 방향성 | 합리적. 단계별 도입 순서 제안에 동의 |
| 현재 단계 | "프롬프트 보강" 단계 진행 중. LangGraph 도입은 시기상조 |
| 비용 우려 | LLM 호출 3~5배 증가 시 비용 부담. 현재 프롬프트 개선으로 충분한지 먼저 검증 필요 |
| 대안 | LangGraph 전에 "2단계 프롬프트(계획 → 작성)" 먼저 시도해볼 만함 |

### 권장 로드맵

1. **현재**: 강화된 프롬프트로 운영하며 품질 모니터링
2. **다음**: 지침 미준수 패턴이 반복되면 2단계 프롬프트 도입
3. **이후**: 그래도 부족하면 LangGraph 파이프라인 구축
