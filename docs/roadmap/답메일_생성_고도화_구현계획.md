# 답메일 생성 고도화 구현 계획

> 작성일: 2026-02-06
> 참고 문서: [답메일_지침_고도화_LangGraph_검토.md](../features/답메일_지침_고도화_LangGraph_검토.md)

---

## 1. 개요

답장 품질을 단계적으로 개선하기 위한 로드맵. LangGraph 도입 전에 프롬프트 개선과 2단계 프롬프트를 먼저 검증한다.

---

## 2. 단계별 구현 계획

### Phase 1: 프롬프트 보강 (완료)

**상태**: ✅ 완료

**적용된 개선 사항**:

| 항목 | 내용 |
|------|------|
| 공식 이름 | 언어별 고정 (춘심이 / ChoonCme / 春心) |
| 노래/춤 구체화 | 발성, 호흡, 비브라토, 웨이브, 팝 등 예시 명시 |
| 질문 지침 | 1~2개 필수, 맥락 기반 질문 예시 확장 |
| 빈 본문 처리 | 이름/메일만으로 감사 표현 가이드 |
| 언어 감지 | 팬레터 언어로 답장 (강조) |

**검증 방법**:
- 실제 답장 발송 후 품질 모니터링
- 지침 미준수 사례 수집 (질문 누락, 언어 불일치 등)

---

### Phase 2: 2단계 프롬프트 (계획 → 작성)

**상태**: 🔜 대기 (Phase 1 검증 후 진행)

**트리거 조건**:
- 지침 미준수 사례가 10% 이상 발생
- 특정 지침(질문, 노래/춤 구체화)이 반복적으로 누락

**구현 내용**:

```
[1단계: 계획 프롬프트]
입력: 팬레터
출력: JSON
  - detected_language: "ko" | "en" | "ja"
  - fan_name: string
  - key_topics: string[]  // 팬레터 핵심 주제
  - emotional_tone: "happy" | "sad" | "neutral" | ...
  - suggested_questions: string[]  // 맥락 기반 질문 후보
  - practice_topic: string  // 이번에 언급할 노래/춤 연습 주제

[2단계: 작성 프롬프트]
입력: 팬레터 + 1단계 계획
출력: JSON { subject, body }
```

**파일 변경**:

| 파일 | 변경 내용 |
|------|----------|
| `lib/llm/reply-prompt.ts` | `buildPlanPrompt()`, `buildWritePrompt()` 추가 |
| `lib/llm/reply-parser.ts` | 계획 JSON 파서 추가 |
| `lib/scheduler/reply-generator.ts` | 2단계 호출 로직 |

**예상 비용**: LLM 호출 1회 → 2회 (약 2배)

---

### Phase 3: LangGraph 파이프라인

**상태**: 📋 계획 (Phase 2로도 부족할 때 진행)

**트리거 조건**:
- 2단계 프롬프트로도 지침 준수율이 낮음
- 조건 분기가 복잡해져 프롬프트 유지보수가 어려움
- 지침 검증/수정 자동화 필요

**파이프라인 설계**:

```
┌─────────────┐
│ 팬레터 입력  │
└──────┬──────┘
       ▼
┌─────────────┐
│ 1. 분석     │ 언어, 감정, 주제 추출
└──────┬──────┘
       ▼
┌─────────────┐
│ 2. 지침 선택 │ 상황별 강조 지침 결정
└──────┬──────┘
       ▼
┌─────────────┐
│ 3. 초안 생성 │
└──────┬──────┘
       ▼
┌─────────────┐
│ 4. 검증     │ 질문 개수, 글자수, 금기어 체크
└──────┬──────┘
       │
   ┌───┴───┐
   │ 통과?  │
   └───┬───┘
    N  │  Y
   ┌───┴───┐
   ▼       ▼
┌─────┐ ┌─────────┐
│수정 │ │ 최종출력 │
└──┬──┘ └─────────┘
   │
   └──→ (4번으로 돌아감)
```

**보완**: 검증 단계에서 미통과 시 수정 후 4번으로 복귀하는 루프는 **최대 재시도 횟수(예: 2~3회)** 를 두어 무한 루프를 방지한다.

**기술 스택**:

| 항목 | 기술 |
|------|------|
| 그래프 | LangGraph (TypeScript) |
| 스키마 | Zod |
| 검증 | 코드 기반 (글자수, 질문 개수 등) |

**구조 제안**:
- 답장 생성을 별도 서비스로 분리
- email-worker는 해당 서비스 호출만 담당

**예상 비용**: LLM 호출 3~5회 (약 3~5배)

## 3. 품질 모니터링 지표

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 질문 포함율 | 답장에 질문 1~2개 포함 여부 | 95% 이상 |
| 언어 일치율 | 팬레터 언어와 답장 언어 일치 | 100% |
| 노래/춤 구체화율 | 구체적 연습 내용 언급 여부 | 90% 이상 |
| 글자수 준수율 | 250자 이상 여부 | 100% |
| 공식 이름 사용율 | 언어별 올바른 이름 사용 | 100% |
